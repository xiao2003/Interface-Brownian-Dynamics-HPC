# Interface Brownian Dynamics HPC

> 语言：MATLAB | 加速方式：MEX (C/C++) | 运行平台：Windows | 文档语言：中文

本项目是一个面向界面布朗动力学研究的 MATLAB 高性能仿真代码库，主要用于模拟单分子在异质界面上的吸附、停留、跳跃、漂移与扩散行为，并对仿真轨迹进行自动化统计分析。当前版本强调两条主线：一条是围绕界面非高斯输运的科学问题建模，另一条是围绕大规模参数扫描的高性能实现。

## 1. 项目定位

本课题关注的不是“如何写一个 MATLAB 脚本”，而是如下科学问题：

- 单分子在异质界面上的运动为何偏离经典高斯扩散模型。
- 表面缺陷、吸附停留时间分布和外加漂移如何共同影响输运统计行为。
- 微观层面的“停留 - 跳跃 - 再吸附”机制，如何映射为宏观层面的位移分布、跳跃长度分布和 MSD 曲线。

为回答这些问题，项目构建了一套界面布朗动力学数值模拟框架。程序允许用户扫描停留时间分布、漂移速度、吸附时间、幂律指数等参数，并自动给出多维统计结果，从而支持后续物理解释、论文写作与答辩展示。

## 2. 当前版本的核心更新

结合当前 `01_Main/JumpingAtMolecularFreq.m` 的实现，本版本相对普通串行脚本有以下核心增强：

- 使用 `parpool + parfeval + fetchNext` 实现异步并行调度，而不是串行等待每个任务执行完成。
- 使用 `parallel.pool.Constant` 共享缺陷地图，显著减少 worker 间的大对象重复传输。
- 为每个重复实验预生成 4 张旋转缺陷地图，用少量基础地图拼接大范围异质界面。
- 在 worker 内部按需动态重建分子周围的 `3x3` 局部缺陷区域，而不是始终维护整张超大地图。
- 在底层跳跃引擎中引入 linked-cell / 空间哈希思想，只搜索局部邻域缺陷点，避免全局暴力距离计算。
- 使用 MEX 版本 `Sub_JumpingBetweenEachFrame_mex_mex.mexw64` 加速核心逐步推进过程。
- 主程序完成仿真后，直接调用轨迹分析模块批量导出 `.mat`、位移分布图、跳跃长度图和 MSD 拟合图。
- 加入动态进度条和 ETA 估计，方便长时间批任务监控。

## 4. 主程序 `JumpingAtMolecularFreq.m` 的工作流

主程序可以分成 6 个步骤理解。

### 4.1 初始化并行环境

程序首先关闭旧并行池，再根据 `feature('numcores')` 创建新的本地并行池，并将核心数限制为 `min(10, feature('numcores'))`。这样做的目的不是简单“占满所有核心”，而是在吞吐量与系统响应之间做平衡，避免桌面环境完全失去响应。

### 4.2 设置物理参数与扫描参数

主程序中给出了以下关键量：

- `t_total`：单次采样时间窗口或总帧时间尺度
- `jf`：分子跳跃频率
- `D`：理论扩散系数
- `adR`：吸附判据半径
- `ds`：缺陷平均间距
- `Ts_list`：相机采样间隔
- `tmads_list`：平均吸附时间
- `TimeIndex_list`：幂律分布指数
- `Xshiftvelocity_list / Yshiftvelocity_list`：外加漂移速度
- `DistributionModes`：停留时间分布类型

其中，

```matlab
k = sqrt(2*D*tau) * 1e9;
```

给出了单步热涨落的特征尺度。这里 `tau = 1/jf`，因此 `k` 可以理解为在分子随机游走过程中，由扩散系数与跳跃时间步共同决定的空间步长标度。

### 4.3 预生成基础缺陷地图

程序没有为整个超大空间显式生成一张全局缺陷图，而是先构造边长为 `L_block = 10000 nm` 的基础区块，并在每个重复实验 `Rep` 下生成：

- `Map1`：原始随机缺陷点集
- `Map2`：旋转 90° 变换图
- `Map3`：旋转 180° 变换图
- `Map4`：旋转 270° 变换图

这样做的目的有两层：

- 用少量基础图样模拟大尺度界面异质性，降低显式建图内存成本。
- 让不同空间区块具有可重复但不完全相同的局部结构，避免“全局严格周期复制”过于理想化。

随后，这些地图被打包为 `MapCell`，再通过 `parallel.pool.Constant` 广播给各个 worker，避免每个任务单独复制大数组。

### 4.4 构造参数任务表

程序将所有参数组合展开成 `Tasks` 矩阵。每一行任务包含：

- 采样时间 `Ts`
- 平均吸附时间 `tm_ads`
- 幂律指数 `TI`
- 漂移速度 `vx, vy`
- 分布类型 `DistMode`
- 重复编号 `Rep`
- 初始位置 `x0, y0`

从工程上看，这一步是标准参数扫描；从科学上看，这一步对应“系统考察物理参数对输运行为的影响”。也就是说，程序不是为单一轨迹服务，而是为统计意义上的输运规律研究服务。

### 4.5 异步并行提交与回收

每个任务通过：

```matlab
futures(i) = parfeval(pool, @Worker_JumpingTask, 1, ...)
```

异步提交。结果不是按提交顺序阻塞等待，而是通过：

```matlab
[idx, res] = fetchNext(futures)
```

按实际完成顺序回收。这样做的优势是：

- 不会因为某个慢任务拖住整个批次。
- 可更充分利用并行池空闲时间。
- 更适合不同参数组合耗时差异明显的仿真任务。

同时，`DataQueue` 用于把每个 worker 的进度回传到主进程，主程序再统一拼装为文本进度条和 ETA。

### 4.6 自动结果归档与后处理

每个任务完成后，程序会：

1. 清洗 NaN 数据。
2. 调用 `Sub_TrajectoryAnalysis` 完成轨迹分析。
3. 基于参数生成唯一子目录名与文件名前缀。
4. 保存 `.mat` 数据包。
5. 导出图像到对应目录。
6. 记录本次运行日志。

文件名中编码了 `Rep`、分布类型、`TI`、`Tads`、`Vx` 等信息，这使得结果目录天然具备“参数可追溯性”。

## 5. Worker 级算法逻辑

`Worker_JumpingTask` 是每个并行任务的真实执行核心。其内部逻辑比普通“逐帧随机游走”更复杂，关键在于动态局部地图与跨帧残余时间的处理。

### 5.1 局部地图而非全局地图

worker 并不在内存里长期维护整个模拟空间，而是维护当前分子附近的局部缺陷图 `XY_l`。当分子位置相对当前局部中心偏移超过阈值 `Th_u` 时，才触发局部地图重建。

### 5.2 3x3 邻域拼接

局部地图重建时，程序先根据分子当前位置求出中心区块索引：

- `Ix_center = floor(cx / L_block)`
- `Iy_center = floor(cy / L_block)`

然后遍历周围 `3x3` 邻域区块，把相应缺陷图转换到全局坐标后拼接起来。这样保证了即使分子接近块边界，也不会因为局部地图截断而漏掉邻近缺陷点。

### 5.3 空间哈希选择地图编号

对于每个局部区块，程序通过：

```matlab
MapIdx = mod(Ix * 73856093 + Iy * 19349663, 4) + 1;
```

将区块坐标映射到 `1..4` 之间的一个编号，从而从四张旋转地图中选取一张。这个做法本质上是一个轻量级空间哈希。其意义在于：

- 同一个空间块始终对应同一张地图，保证可重复性。
- 相邻区块不必使用完全相同的缺陷布局，增强空间异质性。
- 只需保存 4 张基础图样，就能覆盖大范围区域。

## 6. 底层跳跃引擎的算法原理

核心推进函数是 `02_Simulation_Engine/Sub_JumpingBetweenEachFrame_mex.m` 及其编译后的 MEX 版本。它负责在单个采样窗口内，模拟分子的微观跳跃与吸附事件。

### 6.1 运动模型

每一步位移使用近似离散 Langevin / 对流扩散形式：

```matlab
dx = k*randn + vx;
dy = k*randn + vy;
```

其中：

- `k*randn` 表示热涨落导致的随机扩散项。
- `vx, vy` 表示外加漂移或偏置输运项。

从物理上讲，这意味着粒子在每个微观时间步上都同时受到随机热运动和宏观偏向驱动。

### 6.2 吸附判据

程序并不计算复杂势场，而是采用几何接触近似：若分子新位置到最近缺陷点的距离平方小于 `adR^2`，则认为本次发生吸附/停留事件。

这一近似把界面复杂的局域相互作用压缩成“吸附半径”这一有效参数，在保持模型可计算性的同时，保留了表面异质吸附的基本物理意义。

### 6.3 停留时间采样

一旦发生吸附，程序根据 `DistMode` 调用不同的停留时间采样函数：

- `Sub_GeneratePowerLawWithMean`
- `Sub_GenerateExponentialWithMean`
- `Sub_GenerateUniformWithMean`

它们分别代表：

- 幂律分布：强调长停留事件的概率尾部，适合描述更强的异质性或陷阱效应。
- 指数分布：对应更接近泊松型、无记忆的停留过程。
- 均匀分布：可作为对照模型，代表有限范围内相对平坦的停留统计。

从科学问题上看，这部分是整个模型最关键的地方之一，因为界面非高斯输运往往不是由“空间步长异常”主导，而是由“时间停留统计异常”主导。

### 6.4 Linked-cell / 空间哈希搜索

如果对每一步都遍历全部缺陷点计算最近距离，复杂度会非常高。因此底层引擎先把局部缺陷点按照网格单元组织为 linked-cell 结构：

- `head`：每个网格单元对应的链表头
- `list`：缺陷点之间的链式索引

这样做之后，分子新位置只需检查自身所在单元及周围 8 个单元，共 `3x3` 邻域，而不必全局扫描所有缺陷点。算法上等价于把近邻搜索复杂度从近似 `O(N)` 降到局部常数级别。

这也是当前版本能在较大缺陷密度和较高跳跃频率条件下仍保持可运行性的关键技术点。

## 7. 轨迹分析模块的技术细节

`04_Analysis_Modules/Sub_TrajectoryAnalysis.m` 负责把原始定位点转换为具有物理解释的统计量。

### 7.1 同帧点合并与轨迹重建

原始结果先经过 `Sub_MergingLocalizationsInSameFrame` 合并同一帧内的重复点，然后通过 `track.m` 进行轨迹拼接。`track.m` 本质上是经典的粒子追踪算法，通过最小位移原则将离散点连接成连续轨迹。

### 7.2 计算的核心统计量

程序会从轨迹中提取：

- `DX, DY`：相邻步位移分量
- `DL`：步长模长
- `MSD`：均方位移
- `theta, Dphi`：方向变化特征
- `SD`：MSD 线性区斜率对应的扩散尺度指标

这些量对应的科学意义分别是：

- `DX/DY` 用于观察位移分布是否对称、是否受漂移破坏。
- `DL` 用于表征跳跃事件空间尺度。
- `MSD` 用于判断整体是正常扩散、受限扩散还是异常扩散。

### 7.3 MSD 计算与拟合

程序采用时间平均方式构造 `MSD_TA`，并截取前一段线性区间进行拟合。实现上还显式限制：

```matlab
N_MSD = min(10000, size(px_total, 2) - 1)
```

这样做是为了控制内存和计算量，避免长轨迹分析阶段反而成为整个程序的瓶颈。

### 7.4 后台绘图与自动导出

分析模块在执行开始时：

```matlab
set(0, 'DefaultFigureVisible', 'off')
```

强制后台绘图，避免并行环境中大量 figure 弹窗引起的显存和交互问题。最终导出：

- `dx-dy` 二维热力分布
- 一维位移直方图
- 跳跃长度分布
- MSD 拟合曲线

并统一保存到当前参数对应的结果目录下。

## 8. 性能优化思路总结

当前版本的性能提升不是单一技巧，而是多层叠加：

- 并行层：异步任务分发与异步回收
- 数据层：`parallel.pool.Constant` 降低大对象复制成本
- 地图层：用 4 张基础图 + 空间哈希拼接大空间
- 搜索层：linked-cell 局部近邻搜索
- 核心层：MEX 化逐步推进
- 分析层：限制 MSD 长度、后台绘图、批量导出

如果把它概括成一句话，就是：程序用“局部化 + 异步化 + 编译化”的组合策略，把原本代价极高的界面随机输运仿真变成了可批量扫描的工程流程。
